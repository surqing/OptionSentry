# 期权预警系统（OptionSentry）开发文档

> 本文档基于 `KuaiQi / OptionSentry` 项目当前代码（提交于 2026-07 前后）整理，面向开发与维护人员。
> 项目定位：基于 [TqSdk](https://www.shinnytech.com/tqsdk/) 的**期货期权行情监控与预警工具**，只做行情分析和告警，**不包含自动下单、撤单或调仓逻辑**。

---

## 1. 项目概述

OptionSentry 通过 TqSdk 接入期货期权行情，发现合约、编译监控条件、订阅实时行情、计算策略指标，并在指标进入预设区间时输出预警。

- **运行模式**：`live`（实盘行情监控）、`backtest`（历史回测）。
- **交互方式**：命令行（`optionsentry`）+ PyQt6 图形界面（`optionsentry-gui`）。
- **内置策略**：`cp_combo`（CP 组合预警）、`abs_spread`（价差预警）。
- **告警渠道**：文件（JSONL）、邮件（SMTP）、GUI 弹窗与声音。
- **打包**：支持 Windows PyInstaller 打包，输出 exe / zip / SHA256 校验文件。

**关键边界**：当前版本仅支持 `price_basis = "last"`（最新价）；不提供交易执行能力，预警结果需人工核验后再决策。

## 2. 系统架构

系统由一条主链路串起，整体是**「发现 → 编译 → 流式评估 → 状态机告警 → 通知」**的管道：

```
cli.main()
   ├─ load_config(config.toml)         # TOML 解析与校验
   ├─ setup_logging()                  # 滚动日志
   ├─ TqSdkDataSource(config)          # 数据源（实现 MarketDataSource 协议）
   ├─ build_strategy() × N             # 把配置编译为 Strategy 对象
   └─ AlertRunner.run()
         ├─ data_source.discover_universe()   # 合约发现 → Universe
         ├─ _compile_strategies()             # 策略过滤 + 编译 → CompiledStrategy + stream_universe
         └─ for snapshot in data_source.stream(stream_universe):
                ├─ strategy.evaluate(snapshot)        # 生成 ConditionEvaluation 列表
                ├─ alert_engine.process(evaluations) # 边沿触发 → AlertEvent 列表
                └─ notifier.notify(event)            # 通知（邮件/文件/控制台）
```

**核心抽象（位于 `optionsentry/models.py`）**：
- `InstrumentMeta`：合约元数据（标的、行权价、期权方向、到期、限仓等）。
- `Universe`：合约宇宙，提供 `options / futures / underlying_symbols / option_groups / subset / strategy_groups` 等视图。
- `MarketSnapshot`：某一时刻的行情快照（`prices`、`changed_symbols`、`universe`）。
- `ConditionEvaluation`：单条条件的评估结果（`key / active / value / min / max / symbols / message`）。
- `AlertEvent`：一次告警事件（= `timestamp + ConditionEvaluation`）。

## 4. 行情数据流

### 4.1 合约发现 `discover_universe()`
1. 创建 `TqApi`（`_create_api`）。
2. `api.query_quotes(ins_class="OPTION", expired=False, exchange_id=...)` 枚举活跃期权代码。
3. 按 `[universe]` 配置过滤（`all` 全市场 / `指定模式` 的 `only_do` / `not_do`）。
4. 批量查询合约元数据（`_query_metas`，`symbol_info_batch_size` 默认 1000）。
5. 解析标的期货，仅保留「标的为期货」的期权。
6. 应用流动性过滤（`min_volume` / `min_open_interest`，实盘模式）。
7. 返回 `Universe`。若实盘且未触发流动性行情扫描，复用发现阶段的 `TqApi` 连接做订阅。

### 4.2 行情订阅 `stream()`
- **实盘**：`_stream_live` → `_subscribe_live_quotes`，使用 `api.get_quote_list(api_batch)` **分批订阅**（`quote_subscription_batch_size` 默认 1000），每批之间 `api.wait_update` 推进；单连接最大订阅约 `13000` 个标的（`MAX_LIVE_SUBSCRIPTION_SYMBOLS`）。产出 `MarketSnapshot` 流。
- **回测**：按 `strategy_groups()` 分组，每组用 **K 线（duration_seconds=60）** 序列订阅；超过 100 个标的时改用 K 线串行订阅（规避 `get_quote_list` 的 100 标的上限）。回放出同步后的价格快照。

> ⚠️ **已知问题（权限）**：免费 TqSdk 账号对上交所（SSE）/ 深交所（SZSE）期权行情**无访问权限**，其 `get_quote_list` 返回的全部字段为 `NaN`。表现即为订阅成功但行情始终为空。解决方向：升级为付费行情账户，或在 `[universe]` 中过滤掉无权限的交易所（目前 `FUTURE_EXCHANGES = {CFFEX, SHFE, DCE, CZCE, INE, GFEX}`，商品期权多为商品交易所，金融期权在 CFFEX，需按账号权限取舍）。

## 6. 策略引擎

策略以 `[[strategies]]` 数组配置，由 `build_strategy()` 编译为 `Strategy` 对象。

### 6.1 `cp_combo`（CP 组合预警）
在同一标的期货、同一行权年月、同一执行价 K 下，取认购（C）、认沽（P）与标的期货价格 F，计算 **CP 组合偏离率**：

```
deviation = C - P + K - F
value     = deviation / F
```

当 `min_value < value < max_value` 时触发。常见用法：正区间（如 `0.01~inf`）捕捉正向偏离，负区间（`-inf~-0.01`）捕捉负向偏离（正向/负向套利信号）。

### 6.2 `abs_spread`（价差预警）
在同一标的期货、同一行权年月、同一期权方向（CALL/PUT）下，对不同行权价期权**两两组合**，计算**价差比例**：

```
value = (P_first - P_second) / (K_first - K_second)
```

用于侦测同方向不同行权价之间的定价偏离（如盒式、垂直价差异常）。

### 6.3 编译期优化（增量评估）
`strategy.compile(universe)` 在启动期把策略展开为一组**条件**（`_CPComboCondition` / `_AbsSpreadCondition`），并建立 `condition_ids_by_symbol` 倒排索引（标的 → 条件 ID）。运行时只评估 `changed_symbols` 关联的条件（`_affected_condition_ids`），避免每次对全宇宙全扫描，是支撑「千级合约实时评估」的关键。

### 6.4 用户自定义筛选脚本
每个策略可挂 `filter_script`（相对路径基于配置文件目录）与 `filter_function`（默认 `accept`）。脚本需提供：

```python
def accept(option, ctx) -> bool:
    if option.expire_rest_days is None:
        return False
    return 10 <= option.expire_rest_days <= 60   # 仅保留剩余 10~60 天到期期权
```

`FilterContext.underlying(option)` 可拿到对应期货元数据。内置示例：`user_filter_scripts/by_expire_rest_days.py`、`user_filter_scripts/by_expiry.py`。完整字段见 `docs/strategy_filter_accept.md`。

## 7. 预警引擎（AlertEngine）

`alerts.py` 是一个**边沿触发状态机**，避免重复刷屏：

- `_states[key]` 保存每个条件上一拍的 `active` 状态。
- `_should_alert(previous, current)`：
  - `current = False` → 不告警；
  - `previous = False` → 刚进入触发区 → **告警**；
  - `previous = None` 且 `alert_on_first_match = True` → 首拍即触发 → 告警；
  - 其余（已处于触发区）→ 不重复告警。
- `active_count()` 返回当前激活条件数，供日志与 GUI 展示。

## 8. 通知模块（Notifiers）

`notifier = build_notifier(config)`，按 `[notifier.channels]` 组合：

| 通知器 | 行为 |
|---|---|
| `ConsoleNotifier` | 打印到控制台（调试用） |
| `NoOpNotifier` | 空操作（未配置任何渠道时默认） |
| `EmailNotifier` | SMTP 发送，**按 `alert_interval_seconds` 聚合**，失败退避 `failure_backoff_seconds`；邮件含 Plain + HTML 双格式表格 |
| `JsonlAlertRecorder` | 追加写入 `logs/<mode>/alerts.jsonl`（每行一个 JSON） |
| `CompositeNotifier` | 组合多个通知器，任一失败汇总抛出 `NotificationError` |

邮件表格字段：触发时间 / 预警类型（CP组合预警、价差预警）/ 监控对象（标的+到期）/ 方向结构 / 执行价 / 当前指标值 / 预警范围 / 触发条件 / 相关合约。

## 9. 图形界面（GUI）

基于 PyQt6（`optionsentry/gui/`）：
- `app.py`：主窗口，含登录页、配置编辑、启动/停止监控、活跃预警、预警记录、运行日志等页面。
- `credentials.py`：GUI 登录与凭据管理（FEISHU/记忆）。
- `config_store.py`：配置的读写与持久化。
- `runner_adapter.py`：把 `AlertRunner` 放到工作线程，并通过 `RunnerCallbacks`（on_status / on_universe / on_compiled / on_cycle / on_alert）把状态回传到 UI 线程，避免界面卡死。

## 10. 运行模式对比

| 维度 | live（实盘） | backtest（回测） |
|---|---|---|
| 行情来源 | `get_quote_list` 实时订阅 | K 线序列回放（duration_seconds=60） |
| 订阅方式 | 分批（默认 1000/批） | 按 `subscription_batch_size`（默认 50）分批 |
| 时间范围 | 持续运行 | `[backtest] start_dt ~ end_dt` |
| 默认通知 | email + file | file |
| 用途 | 实盘盯盘预警 | 策略区间验证、历史表现 |

## 11. 开发与发布流程

1. **环境**：Python ≥ 3.11；`uv` 包管理；可用 TqSdk 账号。
2. **安装依赖**：`uv sync --dev`
3. **准备配置**：`Copy-Item config.example.toml config.toml`，并设置 `TQSDK_USERNAME` / `TQSDK_PASSWORD` 等环境变量。
4. **运行测试**：`uv run pytest`
5. **运行**：
   - CLI：`uv run optionsentry --config config.toml`
   - GUI：`uv run optionsentry-gui`
6. **提交**：先 `git status --short`、`git diff` 自查，测试通过后提交，提交信息清晰简洁（见 `AGENTS.md`）。
7. **打包**（Windows）：`powershell -NoProfile -ExecutionPolicy Bypass -File .\package.ps1 -StopRunning`，产出 `dist/optionsentry-gui/optionsentry-gui.exe` 与 `dist/OptionSentry-v<version>-windows-x64.zip` + SHA256。
8. **发布**：GitHub Actions `release.yml` 在推送 `v*.*.*` 标签或手动触发时构建；标签须与 `pyproject.toml` 的 `version` 一致（如 `v0.1.1`）。

## 12. 注意事项与已知限制

- `config.toml`、`.env`、日志、打包产物属本地文件，默认不提交。
- 邮件告警需自备 SMTP 服务、发件人、收件人与 `SMTP_PASSWORD`。
- 实盘/回测均依赖 TqSdk 账号权限与行情可用性；**免费账号对 SSE/SZSE 期权行情返回 NaN**（见 §4.2）。
- 当前不提供交易执行能力，预警需人工核验后决策。
- GUI「记住我」会把凭据写入 `config.toml`，仅建议在可信本机使用。

## 13. 后续可扩展点（建议）

- 新增策略类型：继承 `Strategy` / `CompiledStrategy`，在 `build_strategy` 注册。
- 新增数据源：实现 `MarketDataSource` 协议（discover_universe / stream / close），在 `data_sources/__init__.py` 暴露。
- 新增通知渠道：实现 `Notifier.notify`（可选 `flush`），在 `build_notifier` 注册。
- 交易所权限白名单：把「账号实际可用交易所」做成配置，自动剔除 NaN 行情来源。
