## git 版本控制
功能实现之后进行测试和代码审查，经过测试以及审查基本没有问题之后就提交git代码，提交代码时需要注意以下几点：
1. 提交前确保代码已经通过所有测试。
2. 提交信息要清晰、简洁，能够准确描述修改的内容。
3. 避免提交未完成的功能或有明显错误的代码。
4. 定期同步远程仓库的更新，避免产生冲突。

## 新增策略
当用户要求新增、创建或实现期权预警策略，或调整策略专属参数时，必须使用 $create-optionsentry-strategy，并遵循 .agents/skills/create-optionsentry-strategy/SKILL.md 中的实现与验证流程。不要在生产代码中增加旧配置兼容逻辑。

## 界面与代码语言
GUI 中面向用户的标签、按钮、选项和提示尽可能使用中文；配置键、枚举值、类名、字段名以及实现细节使用英文。需要同时满足两者时，GUI 显示中文标签并在底层保存英文值。

## 打包发布
当用户要求打包发布且没有特殊说明时，自动将项目补丁版本号（版本号第三段）递增 1，然后完成测试、打包、产物验证、提交、创建版本标签和发布流程。例如当前版本为 `0.1.2` 时，默认发布 `0.1.3`。
